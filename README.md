# Evidence Scout

A point-of-care clinical reference search tool designed to help floor nurses quickly find evidence-based answers to pressing clinical questions without compromising data privacy.

**Live Demo:** [https://evidence-scout.netlify.app](https://evidence-scout.netlify.app) *(Replace with actual Netlify URL)*

## Tech Stack

- **Framework:** React 18+ with Vite
- **Language:** TypeScript
- **Styling:** Tailwind CSS
- **Routing:** react-router-dom
- **Computer Vision:** OpenCV.js (bundled locally)
- **OCR:** Tesseract.js (running in a Web Worker, bundled locally)
- **Storage:** IndexedDB (via `idb-keyval`) for offline caching of the corpus and WASM binaries
- **PWA:** vite-plugin-pwa for service worker caching

## Local Development

1. Clone the repository and install dependencies:
   ```bash
   npm install
   ```
2. Ensure you have the required external assets and corpus files placed in `/public/` (see instructions below).
3. Start the development server:
   ```bash
   npm run dev
   ```

## Build Instructions

To build the project for production:

```bash
npm run build
```

This will generate the static files in the `dist/` directory, ready to be deployed. The build process does not use server-side rendering.

## Asset Prerequisites

Before running or building the app, you must manually download several large binary files and place them in the `/public/` directory. This ensures the app can function completely offline after the first load.

1. **OpenCV.js:**
   - Download `opencv.js` from [https://docs.opencv.org/4.x/opencv.js](https://docs.opencv.org/4.x/opencv.js)
   - Save it to `public/opencv/opencv.js`

2. **Tesseract.js Assets:**
   - Download `eng.traineddata` from the [tessdata repository](https://github.com/tesseract-ocr/tessdata/raw/main/eng.traineddata)
   - Save it to `public/tesseract/eng.traineddata`
   - Copy `worker.min.js` from `node_modules/tesseract.js/dist/` to `public/tesseract/worker.min.js`
   - Copy `tesseract-core.wasm.js` from `node_modules/tesseract.js-core/` to `public/tesseract/tesseract-core.wasm.js`

## Corpus Update Instructions

The NLP search pipeline requires pre-calculated TF-IDF artifacts. These are generated by the companion Google Colab notebook (`evidence_scout_phase1.ipynb`).

To update the corpus:
1. Run the Google Colab notebook with your updated Markdown corpus in Drive.
2. The notebook will export 4 files: `vocabulary.json`, `idf_values.json`, `document_vectors.json`, and `document_metadata.json`.
3. Download these 4 files from Google Drive and place them in the `public/corpus/` directory of this web app.
4. Restart the web app to load the new corpus.

## Privacy Architecture

**Architecture principle:** After the initial asset and corpus download, all search and image-processing functions run locally in the browser. No user query text, uploaded images, or OCR-extracted content is sent to any backend server or third-party API. This applies the "future is local" thesis — WebAssembly, OpenCV.js, Tesseract.js, and IndexedDB do the heavy lifting client-side.

## Known Limitations

- **Phase 1 Scope:** The corpus is currently limited strictly to upper-extremity orthopedics (shoulder and elbow).
- **OCR Accuracy:** Tesseract running via WebAssembly in the browser is resource-constrained. Poor lighting, blurry photos, or complex document layouts may result in low OCR confidence or garbled text. The app surfaces a warning when confidence is below 60%.
- **Cross-Platform Preprocessing Parity:** To ensure the JavaScript client precisely matches the Python backend export, both pipelines utilize a deliberately simplified preprocessing routine (lowercasing, punctuation stripping, whitespace tokenization, and stopword removal). **Stemming and lemmatization are strictly excluded.** This is an intentional trade-off to guarantee perfect term matching across environments without needing a complex Node.js NLP port.
- **Browser Compatibility (iOS Safari):** iOS Safari imposes strict memory limitations on WebAssembly (often capping around 1GB or less). The app defensively downscales incoming photos to a maximum 1500px edge before passing them to OpenCV and Tesseract to prevent memory crashes.
